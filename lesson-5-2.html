Documentazione Progetto

üìò Introduzione Teorica

Documentazione professionale √® cruciale per successo progetto:

  • README.md: Prima impressione, setup, usage examples
  • Cargo.toml metadata: Keywords, description, license, repository
  • Rustdoc: Commenti documentation inline con esempi
  • CONTRIBUTING.md: Linee guida per contributor
  • CHANGELOG.md: Storico versioni e breaking changes
  • LICENSE: Licenza open source (MIT, Apache 2.0, etc.)
  • Examples: Esempi nel folder examples/

Buona documentazione attrae utenti, facilita onboarding, e costruisce reputazione di qualit√†.

// ========================================

/// Rappresenta un editor di testo semplice
///
/// # Fields
/// * \`content\` - Il testo attuale
/// * \`file_path\` - Percorso file se salvato
///
/// # Example
/// \`\`\`
/// let mut editor = Editor::new();
/// editor.add_char('h');
/// assert_eq!(editor.get_content(), "h");
/// \`\`\`
#[derive(Debug)]
pub struct Editor {
    content: String,
    file_path: Option,
}

impl Editor {
    /// Crea un nuovo editor vuoto
    ///
    /// # Examples
    /// \`\`\`
    /// let editor = Editor::new();
    /// assert_eq!(editor.get_content(), "");
    /// \`\`\`
    pub fn new() -> Self {
        Editor {
            content: String::new(),
            file_path: None,
        }
    }

    /// Carica file da percorso
    ///
    /// # Arguments
    /// * \`path\` - Percorso del file da aprire
    ///
    /// # Returns
    /// * \`Ok(())\` se successo
    /// * \`Err(std::io::Error)\` se fallisce
    ///
    /// # Errors
    /// Ritorna errore se file non leggibile o non esiste
    ///
    /// # Examples
    /// \`\`\`no_run
    /// let mut editor = Editor::new();
    /// editor.load_file("hello.txt")?;
    /// # Ok::<(), std::io::Error>(())
    /// \`\`\`
    pub fn load_file(&mut self, path: &str) -> std::io::Result<()> {
        self.content = std::fs::read_to_string(path)?;
        self.file_path = Some(path.to_string());
        Ok(())
    }

    /// Ritorna il contenuto corrente
    pub fn get_content(&self) -> &str {
        &self.content
    }

    /// Aggiunge un carattere
    pub fn add_char(&mut self, c: char) {
        self.content.push(c);
    }
}

‚úèÔ∏è Esercizi Pratici

  1. README Professionale

    Scrivi README con: descrizione, features, installation, usage, examples, license.

  2. Documenta il Codice

    Aggiungi rustdoc a tutte le funzioni pubbliche con description, examples, errors.

  3. CONTRIBUTING.md

    Crea linee guida per contributor: setup, stile, testing, processo PR.

  4. CHANGELOG.md

    Documenta versioni: v0.1.0, v0.2.0, v1.0.0 con Added/Fixed/Changed.

  5. Cargo.toml Metadata

    Aggiungi: description, keywords, categories, license, repository, homepage.

  6. Challenge: Docs Complete

    Documenta libreria intera, genera con cargo doc --open, pubblica su docs.rs.

‚ö†Ô∏è Troubleshooting

‚ùå Cargo doc genera errori

Causa: Errori sintassi rustdoc o broken links.

Soluzione: Verifica sintassi, ordine tag (/// prima di pub), link validi.

‚ùå Documentazione non aggiorna

Soluzione: cargo clean && cargo doc --open

‚ùå Examples non compilano

Soluzione: Aggiungi \`\`\`no_run se esempio non pu√≤ girare standalone.

üí° Tip: Doc Tests

Rust esegue gli esempi in rustdoc! Assicurati compilino con cargo test --doc

üí° Tip: Link in Docs

Usa \`[Nome](crate::module::function)\` per link interni tra docs.
