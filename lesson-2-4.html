Crate e Gestione delle Dipendenze

üìò Introduzione Teorica

Il sistema di gestione dipendenze di Rust si basa su Cargo e crates.io, il registry ufficiale delle librerie Rust. Un crate √® un'unit√† di compilazione: pu√≤ essere una libreria o un binario.

  • Cargo.toml: File di configurazione che specifica dipendenze, metadati e configurazioni di build
  • crates.io: Repository centrale con oltre 100.000 crate disponibili
  • Semantic Versioning: Gestione delle versioni con formato MAJOR.MINOR.PATCH
  • Features: Abilita/disabilita parti opzionali delle dipendenze
  • Cargo.lock: File generato automaticamente che fissa le versioni esatte usate

Cargo gestisce automaticamente download, compilazione e aggiornamento delle dipendenze, semplificando notevolmente lo sviluppo.

üîß Procedura Passo-Passo

  1. Aggiungere una dipendenza base

    Modifica Cargo.toml:

    [package]
name = "mio_progetto"
version = "0.1.0"
edition = "2021"

[dependencies]
serde = "1.0"           # Serialization
rand = "0.8"            # Numeri random
chrono = "0.4"          # Date e timestamp

    Esegui cargo build per scaricare e compilare le dipendenze.

  2. Usare una crate nel codice 
    // src/main.rs
use rand::Rng;  // Importa il trait Rng
use chrono::Local;  // Importa utilit√† per date

fn main() {
    // Usa rand
    let numero = rand::thread_rng().gen_range(1..=100);
    println!("Numero casuale: {}", numero);

    // Usa chrono
    let ora = Local::now();
    println!("Ora attuale: {}", ora.format("%Y-%m-%d %H:%M:%S"));

  3. Specificare versioni 
    [dependencies]
# Versione esatta
serde = "=1.0.150"

# Versione minima (>= 1.0.0, < 2.0.0)
serde = "1.0"

# Wildcard
serde = "1.*"

# Range
serde = ">=1.0.0, <1.5.0"

# Git repository
mia_lib = { git = "https://github.com/user/mia_lib" }

# Percorso locale
mia_lib_locale = { path = "../mia_lib" }
  4. Features opzionali 
    [dependencies]
# Solo features necessarie
serde = { version = "1.0", features = ["derive"] }

# Disabilita features di default
tokio = { version = "1.0", default-features = false, features = ["rt"] }
  5. Dipendenze di sviluppo 
    # Solo per test e bench, non incluse nel binario finale
[dev-dependencies]
criterion = "0.5"  # Per benchmark
mockito = "1.0"    # Per mock nei test
  6. Profili di compilazione 
    [profile.dev]
opt-level = 0  # Nessuna ottimizzazione (veloce da compilare)

[profile.release]
opt-level = 3         # Massima ottimizzazione
lto = true            # Link-Time Optimization
codegen-units = 1     # Singola unit√† (pi√π lento ma pi√π ottimizzato)
strip = true          # Rimuovi simboli debug

// ESEMPIO 1: Uso di serde per JSON
// ========================================
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize, Debug)]
struct Person {
    name: String,
    age: u32,
    email: String,
}

fn esempio_serde() {
    // Serializzazione: struct -> JSON
    let person = Person {
        name: "Mario Rossi".to_string(),
        age: 30,
        email: "mario@example.com".to_string(),
    };

    let json = serde_json::to_string(&person).unwrap();
    println!("JSON: {}", json);

    // Deserializzazione: JSON -> struct
    let json_str = r#"{"name":"Luigi Verdi","age":25,"email":"luigi@example.com"}"#;
    let person: Person = serde_json::from_str(json_str).unwrap();
    println!("Person: {:?}", person);
}

use clap::Parser;

#[derive(Parser)]

    /// Nome da salutare
    #[arg(short, long)]
    name: String,

    /// Numero di ripetizioni
    #[arg(short, long, default_value_t = 1)]
    count: u8,
}

fn esempio_clap() {
    let args = Args::parse();

    for _ in 0..args.count {
        println!("Ciao, {}!", args.name);
    }
}

// ESEMPIO 4: Numeri random con rand
// ========================================
use rand::Rng;

fn esempio_rand() {
    let mut rng = rand::thread_rng();

    // Numero tra 1 e 100
    let n: i32 = rng.gen_range(1..=100);
    println!("Numero casuale: {}", n);

    // Booleano random
    let b: bool = rng.gen();
    println!("Booleano: {}", b);

    // Shuffle di un vettore
    use rand::seq::SliceRandom;
    let mut numeri = vec![1, 2, 3, 4, 5];
    numeri.shuffle(&mut rng);
    println!("Shuffled: {:?}", numeri);
}

// ESEMPIO 5: Date con chrono
// ========================================
use chrono::{DateTime, Local, Duration};

fn esempio_chrono() {
    // Data e ora corrente
    let ora = Local::now();
    println!("Ora: {}", ora);

    // Formattazione
    println!("Formattata: {}", ora.format("%Y-%m-%d %H:%M:%S"));

    // Operazioni sulle date
    let domani = ora + Duration::days(1);
    println!("Domani: {}", domani);

    // Parsing di una stringa
    use chrono::NaiveDate;
    let data = NaiveDate::parse_from_str("2024-12-25", "%Y-%m-%d").unwrap();
    println!("Natale: {}", data);
}

// ESEMPIO 6: SQLite con rusqlite
// ========================================
use rusqlite::{Connection, Result as SqlResult};

#[derive(Debug)]
struct User {
    id: i32,
    name: String,
    email: String,
}

fn esempio_sqlite() -> SqlResult<()> {
    // Apri/crea database
    let conn = Connection::open("mio_db.db")?;

    // Crea tabella
    conn.execute(
        "CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY,
            name TEXT NOT NULL,
            email TEXT NOT NULL
        )",
        [],
    )?;

    // Insert
    conn.execute(
        "INSERT INTO users (name, email) VALUES (?1, ?2)",
        ["Mario", "mario@example.com"],
    )?;

    // Query
    let mut stmt = conn.prepare("SELECT id, name, email FROM users")?;
    let users = stmt.query_map([], |row| {
        Ok(User {
            id: row.get(0)?,
            name: row.get(1)?,
            email: row.get(2)?,
        })
    })?;

    for user in users {
        println!("User: {:?}", user?);
    }

    Ok(())
}

    Ok(())
}

‚ö†Ô∏è Troubleshooting

‚ùå Errore: "could not find Cargo.toml"

Causa: Non sei nella directory del progetto Cargo.

Soluzione:

# Naviga nella directory del progetto
cd mio_progetto

# Verifica che Cargo.toml esista
ls Cargo.toml

‚ùå Errore: "failed to parse manifest"

Causa: Sintassi errata in Cargo.toml.

Soluzione:

  • Verifica l'indentazione (usa spazi, non tab)
  • Controlla le virgolette
  • Usa cargo check per vedere l'errore dettagliato

‚ùå Errore: "version cannot be empty"

Causa: Versione della dipendenza non specificata correttamente.

Soluzione:

# SBAGLIATO:
[dependencies]
serde = ""

# CORRETTO:
[dependencies]
serde = "1.0"

‚ùå Errore: "no matching version found"

Causa: La versione richiesta non esiste su crates.io.

Soluzione:

  • Cerca su crates.io la versione corretta
  • Usa cargo search nome_crate
  • Controlla se il nome √® scritto correttamente

‚ùå Compilazione molto lenta

Causa: Troppe dipendenze o compilazione da zero.

Soluzioni:

  • Usa cargo check invece di cargo build durante sviluppo
  • Abilita cache con sccache
  • Disabilita features non necessarie
  • Compila in modalit√† incrementale (default in dev)
