Crate e Gestione Dipendenze

ğŸ“˜ Introduzione Teorica

Cargo gestisce dipendenze e versioning:

  • Cargo.toml: Configurazione progetto
  • crates.io: Repository pubblico
  • Semantic versioning: Major.Minor.Patch
  • Features: Opzioni compilazione

ğŸ’» Codice Sorgente Commentato

// Cargo.toml
[package]
name = "my_app"
version = "0.1.0"
edition = "2021"

[dependencies]
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
rand = "0.8"
tokio = { version = "1", features = ["full"] }

// main.rs
use rand::Rng;
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize)]
struct User {
    name: String,
    age: u32,
}

fn main() {
    let n = rand::thread_rng().gen_range(1..=100);
    println!("Random: {}", n);

    let user = User {
        name: "Mario".to_string(),
        age: 30,
    };

    let json = serde_json::to_string(&user).unwrap();
    println!("{}", json);
}

  2. rand: Genera numeri casuali 1-100.
  3. tokio: Async/await hello world.
  4. Features: Abilita features opzionali.
