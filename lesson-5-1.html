Repository GitHub

üìò Introduzione Teorica

GitHub √® essenziale per developer professionali Rust:

  • Git fundamentals: init, add, commit, push, pull, branch, merge
  • Repository structure: README, LICENSE, .gitignore, docs/
  • Branching strategy: main, develop, feature branches
  • Pull requests: Code review, discussioni, approval workflow
  • Releases & tags: Semantic versioning (major.minor.patch)
  • GitHub Actions: CI/CD automatico (test, build, deploy)
  • Collaboration: Issues, projects, milestones

Padroneggiare GitHub √® fondamentale per carriera in software development, open source, e team collaboration.

üîß Procedura Passo-Passo

  1. Setup Git locale 
    git config --global user.name "Mario Rossi"
git config --global user.email "mario@example.com"
cd progetto
git init
  2. Crea repository su GitHub

    Dashboard ‚Üí New repository ‚Üí Public/Private ‚Üí Create

  3. Connetti locale a remoto 
    git remote add origin https://github.com/user/repo.git
git branch -M main
git add .
git commit -m "Initial commit"
git push -u origin main
  4. Workflow branching completo 
    git checkout -b feature/nome-feature
# Modifica file...
git add .
git commit -m "feat: descrizione feature"
git push origin feature/nome-feature
# Crea Pull Request su GitHub
# Merge dopo approvazione
git checkout main
git pull origin main
git merge feature/nome-feature
git push origin main
  5. SSH authentication setup 
    ssh-keygen -t ed25519 -C "email@example.com"
cat ~/.ssh/id_ed25519.pub
# Copia in GitHub ‚Üí Settings ‚Üí SSH keys
  6. Tagging e releases 
    git tag -a v1.0.0 -m "Release version 1.0.0"
git push origin v1.0.0
# Crea release su GitHub con changelog

üíª Codice Sorgente Commentato

// ========================================
// GIT WORKFLOW COMPLETO
// ========================================

// SETUP INIZIALE
$ git config --global user.name "Mario Rossi"
$ git config --global user.email "mario@example.com"
$ git config --global init.defaultBranch main

// CREA REPOSITORY LOCALE
$ cd mio_progetto
$ git init
$ git add .
$ git commit -m "Initial commit"

// ========================================
// BRANCHING WORKFLOW FEATURE BRANCH
// ========================================

// Crea feature branch
$ git checkout -b feature/add-search

// Modifica file, aggiungi feature
$ echo "pub fn search()" >> src/lib.rs

// Commit con messaggio semantico
$ git add src/lib.rs
$ git commit -m "feat: add search functionality"

// Push branch su GitHub
$ git push origin feature/add-search

// ‚Üí Su GitHub: crea Pull Request
// ‚Üí Chiedi review, discussion
// ‚Üí Merge dopo approvazione

// Torna su main e aggiorna locale
$ git checkout main
$ git pull origin main

// Merge feature in main
$ git merge feature/add-search

// Elimina branch locale
$ git branch -d feature/add-search

// Elimina branch remoto
$ git push origin --delete feature/add-search

// ========================================
// COMANDI ESSENZIALI GIT
// ========================================

$ git status              # Mostra stato working tree
$ git diff                # Differenze non staged
$ git diff --staged       # Differenze staged
$ git log --oneline       # History compatto
$ git log --graph --all   # Visualizza branches
$ git reset --soft HEAD~1 # Annulla commit, keep changes
$ git reset --hard HEAD~1 # Annulla commit completamente
$ git revert HEAD         # Crea commit che annulla
$ git stash               # Salva temporaneamente modifiche
$ git cherry-pick abc123  # Applica commit specifico

// ========================================
// TAGGING E RELEASES
// ========================================

// Crea tag annotato (per releases)
$ git tag -a v1.0.0 -m "Release version 1.0.0"

// Crea tag lightweight
$ git tag v1.0.0-beta

// Lista tag
$ git tag -l

// Mostra info tag
$ git show v1.0.0

// Push tag singolo
$ git push origin v1.0.0

// Push tutti tag
$ git push origin --tags

// ========================================
// GITHUB ACTIONS CI/CD
// ========================================

// File: .github/workflows/rust.yml
name: Rust CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Install Rust
      uses: dtolnay/rust-toolchain@stable
    - name: Build
      run: cargo build --verbose
    - name: Run tests
      run: cargo test --verbose
    - name: Run clippy
      run: cargo clippy -- -D warnings
    - name: Check formatting
      run: cargo fmt -- --check

‚ö†Ô∏è Troubleshooting

‚ùå "permission denied (publickey)"

Causa: SSH key non configurata o non aggiunta a GitHub.

Soluzione: Genera SSH key e aggiungi a GitHub Settings ‚Üí SSH keys.

‚ùå "Your branch is ahead of origin"

Causa: Commit locale non pushati.

Soluzione: git push origin branch_name

‚ùå Merge conflict

Causa: Stesso file modificato in due branch.

Soluzione: Edita file, risolvi conflitti, git add . && git commit

‚ùå "failed to push"

Soluzione: git pull prima di push se remoto √® avanti.

üí° Tip: Commit Amend

git commit --amend modifica ultimo commit senza creare nuovo.

üí° Tip: Revert Accidentale

git revert HEAD crea nuovo commit che annulla l'ultimo.
