Repository GitHub

üìò Introduzione Teorica

GitHub √® il platform principale per versionare e collaborare:

  • Git fundamentals: Commit, push, pull, branch, merge
  • Repository structure: README, LICENSE, .gitignore
  • Branching strategy: main, develop, feature branches
  • Pull requests: Code review workflow
  • Releases & tags: Semantic versioning
  • GitHub Actions: CI/CD automatica

Padroneggiare GitHub √® essenziale per developer professionisti.

üîß Procedura Passo-Passo

  1. Setup Git locale 
    git config --global user.name "Nome"
git config --global user.email "email@example.com"
cd progetto && git init
  2. Crea repository su GitHub

    Dashboard ‚Üí New repository ‚Üí scegli Public/Private

  3. Connetti locale a remoto 
    git remote add origin https://github.com/user/repo.git
git branch -M main
git push -u origin main
  4. Workflow branching 
    git checkout -b feature/nome-feature
# Lavora e commit
git add . && git commit -m "feat: descrizione"
git push origin feature/nome-feature
# Crea Pull Request su GitHub
  5. Setup SSH per autenticazione 
    ssh-keygen -t ed25519 -C "email@example.com"
cat ~/.ssh/id_ed25519.pub  # Copia in GitHub Settings

üíª Codice Sorgente Commentato

// ========================================
// GIT WORKFLOW COMPLETO
// ========================================

// Setup iniziale
$ git config --global user.name "Mario Rossi"
$ git config --global user.email "mario@example.com"

// Crea repository locale
$ cd mio_progetto
$ git init
$ git add .
$ git commit -m "Initial commit"

// ========================================
// BRANCHING WORKFLOW
// ========================================

// Feature branch
$ git checkout -b feature/add-search
$ git add src/search.rs
$ git commit -m "feat: add search functionality"
$ git push origin feature/add-search
// ‚Üí Crea Pull Request su GitHub

// Merge dopo approvazione
$ git checkout main
$ git pull origin main
$ git merge feature/add-search
$ git push origin main
$ git branch -d feature/add-search

// ========================================
// COMANDI ESSENZIALI
// ========================================

$ git status          # Mostra stato working tree
$ git log --oneline   # History compatto
$ git diff            # Differenze non staged
$ git reset --soft HEAD~1  # Annulla commit, keep changes
$ git revert HEAD     # Crea commit che annulla
$ git stash           # Salva temporaneamente modifiche
$ git cherry-pick abc123  # Applica commit specifico

// ========================================
// TAGS PER RELEASES
// ========================================

$ git tag -a v1.0.0 -m "Release version 1.0.0"
$ git push origin v1.0.0

$ git tag -l          # Lista tag

// ========================================
// .gitignore TEMPLATE
// ========================================

target/
Cargo.lock
*.swp
.DS_Store
.vscode/
.env
node_modules/

// ========================================
// SEMANTIC COMMIT MESSAGES
// ========================================

feat(parser): add new parsing algorithm
fix(ui): resolve button alignment issue
docs(readme): update installation instructions
refactor(network): simplify connection logic
test(utils): add unit tests for helpers

// ========================================
// GITHUB ACTIONS CI/CD
// ========================================

// File: .github/workflows/rust.yml
name: Rust CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Build
      run: cargo build --release
    - name: Run tests
      run: cargo test
    - name: Run clippy
      run: cargo clippy -- -D warnings

‚úèÔ∏è Esercizi Pratici

  1. Setup GitHub Base

    Crea account GitHub, nuovo repository pubblico, push primo commit.

  2. Branching Workflow

    Crea 3 feature branch, aggiungi commit, merge su main visualizzando history.

  3. Pull Request

    Fork progetto open source, crea PR con piccola modifica/fix.

  4. Tagging & Releases

    Crea tag v1.0.0, pubblica release con changelog su GitHub.

  5. .gitignore Ottimale

    Configura .gitignore per Rust con tutti i file da escludere.

  6. Challenge: GitHub Actions

    Setup CI/CD che compila, testa e lancia clippy automaticamente.
