Repository GitHub

üìò Introduzione Teorica

GitHub √® il principale platform per versionare, collaborare e pubblicare progetti open source. Un buon repository attrae contributors e dimostra professionismo.

  • Git fundamentals: Commit, push, pull, branch, merge
  • Repository structure: README, LICENSE, .gitignore, src/
  • Branching strategy: main, develop, feature branches
  • Pull requests: Code review e collaborazione
  • Issues & labels: Tracciamento bug e features
  • Releases & tags: Versioni pubbliche e semantic versioning
  • GitHub Actions: CI/CD automatica

Padroneggiare GitHub √® essenziale per ogni developer professionista.

üîß Procedura Passo-Passo

  1. Crea un account GitHub

    Visita github.com e registrati con email.

  2. Crea repository locale 
    # Naviga nella directory progetto
cd mio_progetto

# Inizializza git
git init

# Aggiungi file
git add .

# Commit iniziale
git commit -m "Initial commit"
  3. Crea repository su GitHub

    Dashboard ‚Üí New repository ‚Üí Scegli nome, descrizione, visibility (Public/Private)

  4. Connetti locale a remoto 
    # Aggiungi remoto (sostituisci username/repo)
git remote add origin https://github.com/username/repo.git

# Push verso GitHub
git branch -M main
git push -u origin main
  5. Configura .gitignore 
    # File: .gitignore
target/
*.swp
*.swo
.DS_Store
Cargo.lock
  6. Setup SSH per autenticazione 
    # Genera chiave SSH
ssh-keygen -t ed25519 -C "email@example.com"

# Copia public key
cat ~/.ssh/id_ed25519.pub

# Aggiungi in GitHub Settings ‚Üí SSH keys

üíª Codice Sorgente Commentato

// ========================================
// GIT WORKFLOW COMPLETO
// ========================================

// ========================================
// 1. SETUP INIZIALE
// ========================================

$ git config --global user.name "Il Tuo Nome"
$ git config --global user.email "email@example.com"

$ cd mio_progetto
$ git init
$ git add .
$ git commit -m "Initial commit: struttura base progetto"

// ========================================
// 2. LAVORARE CON BRANCHES
// ========================================

// Crea feature branch
$ git checkout -b feature/add-search

// Lavora e commit
$ git add src/search.rs
$ git commit -m "Add search functionality"
$ git commit -m "Add tests for search"

// Torna a main e merge
$ git checkout main
$ git pull origin main
$ git merge feature/add-search

// Push verso remoto
$ git push origin main

// Elimina branch locale
$ git branch -d feature/add-search
$ git push origin --delete feature/add-search

// ========================================
// 3. WORKFLOW COLLABORATIVO
// ========================================

// Clona un repository
$ git clone https://github.com/user/project.git
$ cd project

// Crea branch locale
$ git checkout -b fix/issue-123

// Fai modifiche
$ git status                    # Vedi cosa √® cambiato
$ git diff src/main.rs         # Vedi differenze file specifico
$ git add src/main.rs
$ git commit -m "Fix: resolve issue #123"

// Push
$ git push origin fix/issue-123

// Crea Pull Request su GitHub (via web interface)

// Dopo merge su GitHub
$ git checkout main
$ git pull origin main

// ========================================
// 4. REVERT E UNDO
// ========================================

// Annulla ultimo commit (ma keep changes)
$ git reset --soft HEAD~1

// Annulla ultimo commit (discard changes)
$ git reset --hard HEAD~1

// Vedi history
$ git log --oneline -10

// Revert un commit specifico
$ git revert abc123

// Cherry-pick commit da un branch
$ git cherry-pick abc123

// ========================================
// 5. TAGGING E RELEASES
// ========================================

// Crea tag per versione
$ git tag -a v1.0.0 -m "Release version 1.0.0"
$ git push origin v1.0.0

// Lista tag
$ git tag -l

// Crea tag leggero
$ git tag v1.0.1

// ========================================
// 6. STASHING (TEMPORANEO)
// ========================================

// Salva temporaneamente modifiche
$ git stash

// Torna al lavoro precedente
$ git stash pop

// Elenca stash
$ git stash list

// ========================================
// 7. REBASING (ADVANCED)
// ========================================

// Riscrivi history (MAI su branch pubblici!)
$ git rebase -i HEAD~3

// Usa "pick", "reword", "squash" per modificare commit

‚úèÔ∏è Esercizi Pratici

  1. Setup GitHub Base

    Crea account GitHub (se non esiste), crea repository pubblico e pushalo prima commit.

  2. Git Workflow Locale

    Crea 3 branch feature, aggiungi commit, merge su main, visualizza history con git log.

  3. Collaborazione Simulata

    Clona un repository open source, crea branch, fai modifica (anche piccola), crea PR.

  4. Tagging e Releases

    Crea tag v1.0.0 nel tuo repository, crea GitHub Release con changelog.

  5. .gitignore Ottimale

    Crea .gitignore per progetto Rust con tutti i file da escludere, verifica con git check-ignore.

  6. Challenge: GitHub Actions CI/CD

    Setup GitHub Actions per:

    • Compilare il progetto
    • Eseguire test
    • Eseguire clippy linter
    • Creare badge status nel README

‚ùå Errore "permission denied (publickey)"

Causa: SSH key non configurata correttamente.

Soluzione:

‚ùå "fatal: not a git repository"

Soluzione: git init per inizializzare

‚ùå "Your branch is ahead of 'origin/main'"

Causa: Hai commit locali non pushati.

Soluzione: git push origin main
