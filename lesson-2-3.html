Struttura di un Progetto

üìò Introduzione Teorica

Organizzare codice in moduli:

  • main.rs: Entry point
  • lib.rs: Root libreria
  • mod: Dichiarazione modulo
  • pub: Esportare pubblicamente

üîß Procedura Passo-Passo

  1. Crea modulo 
    // src/utils.rs
pub fn saluta() {
    println!("Ciao!");
}

// src/main.rs
mod utils;

fn main() {
    utils::saluta();
}

üíª Codice Sorgente Commentato

// src/lib.rs
pub mod utils;
pub mod network;

// src/utils.rs
pub fn helper() {
    println!("Helper!");
}

pub fn greet(name: &str) {
    println!("Ciao, {}!", name);
}

// src/main.rs
use my_project::utils;

fn main() {
    utils::helper();
    utils::greet("Mario");
}

// SOTTOCARTELLE
// src/network/mod.rs
pub mod client;
pub mod server;

// src/network/client.rs
pub fn connect() {
    println!("Connecting...");
}

  1. Modulo math: Add/subtract/multiply.
  2. Sottocartelle: network/client.rs.
  3. Moduli annidati: Struttura complessa.
